# PART C: THE UPGRADE (Agentic + From the Future)

## C.1 Design Philosophy

FaceMortgage will feel like:
- **Evidence-first**: Every LO stat is NMLS-verified or source-labeled
- **Explainable**: "Why this LO?" has a clear answer
- **Agentic but bounded**: AI assists, never decides
- **Future-proof**: Modular architecture for AI model upgrades

## C.2 Required Capabilities

### 1. TurboTax-Style Intake DNA

```yaml
Schema-Driven Questionnaire:
  - loan_purpose: ["purchase", "refinance", "heloc"]
  - property_type: ["single_family", "condo", "multi_unit"]
  - timeline: ["immediate", "30_days", "exploring"]
  - special_needs: ["self_employed", "first_time", "jumbo", "va_eligible"]
  
Contradiction Detection:
  - IF loan_purpose=refinance AND property_type=null THEN prompt for current property
  - IF va_eligible=true AND veteran_status=false THEN clarify

Uncertainty Paths:
  - "I'm not sure" → adds to verification checklist + LO can clarify on call
```

### 2. Scenario Universe Builder

```
Borrower Profile → Eligible LOs → Ranked by Match Score
                 ↓
         [Never Invented Data]
         - Only show LOs licensed in borrower's state
         - Only show loan types LO actually offers
         - Label unknowns: "Rate estimate - varies by credit"
```

### 3. Blockers/Unlockers Engine

- 🟢 Quick Win (24h): Pull credit report

## D.6 Module Specifications

### Module: Matching Algorithm

**Purpose**: Rank LOs by borrower fit score

**Inputs**:
```python
class BorrowerProfile(BaseModel):
    state: str  # Required - determines LO licensing
    loan_purpose: LoanPurpose
    property_type: PropertyType
    timeline: Timeline
    special_needs: List[SpecialNeed]
    preferred_language: Optional[str]
```

**Outputs**:
```python
class LOMatch(BaseModel):
    lo_id: UUID
    match_score: float  # 0-100
    match_reasons: List[str]  # "Licensed in CA", "FHA specialist"
    availability: AvailabilityStatus
    response_time_seconds: float
```

**Schema**:
```sql
CREATE TABLE match_logs (
    id UUID PRIMARY KEY,
    borrower_session_id UUID NOT NULL,
    algorithm_version VARCHAR(20) NOT NULL,
    input_hash VARCHAR(64) NOT NULL,
    top_matches JSONB NOT NULL,
    created_at TIMESTAMP DEFAULT now()
);
```

**Dependencies**: NMLS service, Presence service

**Failure Modes**:
- No licensed LOs in state → Show "No results" + suggest nearby states
- High load → Cache recent matches for similar profiles
- NMLS API down → Use cached credentials (max 24h stale)

**Acceptance Tests**:
1. Borrower in CA sees only CA-licensed LOs
2. VA-eligible borrower sees VA specialists ranked higher
3. Match reasons are human-readable
4. Algorithm version logged with every match

### Module: Presence Service

**Purpose**: Track real-time LO availability

**Inputs**: WebSocket heartbeats, explicit status changes

**Outputs**:
```python
class PresenceStatus(BaseModel):
    lo_id: UUID
    status: Literal["online", "busy", "offline"]
    last_seen: datetime
    avg_pickup_seconds: float
```

**Failure Modes**:
- WebSocket disconnect → Mark offline after 30s timeout
- Redis unavailable → Fallback to database (degraded performance)

## D.7 Delegation Notes for Smaller Models

When delegating tasks to smaller coding models:

1. **Always provide**: Schema definitions, acceptance tests, example inputs/outputs
2. **Never assume**: Model knows business context - explain why, not just what
3. **Chunk appropriately**: One endpoint per prompt, one component per prompt
4. **Validate outputs**: Run acceptance tests before human review
